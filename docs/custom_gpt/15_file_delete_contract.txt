# 업로드 파일 삭제 v11

사용자 화면:
- `/sources/manage`

기능:
- 업로드된 source 목록 조회
- subject 필터
- 개별 Delete 버튼
- 체크박스 선택 후 선택 삭제
- 전체 선택 후 일괄 삭제
- source record 삭제
- source_chunks 삭제
- 로컬 업로드 파일 삭제 시도

Action:
- deleteSource

사이트 UI:
- 여러 파일을 한 번에 지울 때는 `/sources/manage`에서 체크박스로 선택 후 `선택 삭제`를 사용한다.
- Custom GPT Action으로 대량 삭제를 자동 실행하지 않는다. 삭제 대상이 여러 개인 경우 사용자가 사이트 UI에서 직접 확인 후 삭제하도록 안내한다.

GPT 사용 규칙:
사용자가 "파일 삭제", "자료 삭제", "업로드 파일 지워줘"라고 하면 먼저 getMappingStatus 또는 /sources 목록을 확인하고 source_id를 특정한다.
source_id가 명확할 때만 deleteSource를 호출한다.
애매하면 삭제할 자료 제목 또는 source_id를 짧게 확인한다.

삭제 후:
- 삭제된 title
- deleted_source_id
- deleted_chunks
- deleted_file 여부
를 알려준다.

주의:
unit_map 내부의 과거 참조는 자동으로 지우지 않는다.
삭제 후 getMappingStatus에서 deletedOrMissingReferences로 표시될 수 있다.
