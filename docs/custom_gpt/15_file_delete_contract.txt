# 업로드 파일 삭제 v11

사용자 화면:
- `/sources/manage`

기능:
- 업로드된 source 목록 조회
- subject 필터
- Delete 버튼
- source record 삭제
- source_chunks 삭제
- 로컬 업로드 파일 삭제 시도

Action:
- deleteSource

GPT 사용 규칙:
사용자가 "파일 삭제", "자료 삭제", "업로드 파일 지워줘"라고 하면 먼저 getMappingStatus 또는 /sources 목록을 확인하고 source_id를 특정한다.
source_id가 명확할 때만 deleteSource를 호출한다.
애매하면 삭제할 자료 제목 또는 source_id를 짧게 확인한다.

삭제 후:
- 삭제된 title
- deleted_source_id
- deleted_chunks
- deleted_file 여부
를 알려준다.

주의:
unit_map 내부의 과거 참조는 자동으로 지우지 않는다.
삭제 후 getMappingStatus에서 deletedOrMissingReferences로 표시될 수 있다.
