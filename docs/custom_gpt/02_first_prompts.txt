# 1. 강의 정리본 생성 시작

저장된 자료를 기준으로 강의 정리본을 만들어줘.

먼저 searchSources로 transcript, lecture_slides, textbook, exam_trend 관련 chunk를 찾아보고,
createProject(project_type="lecture_note")를 호출해서 프로젝트를 만든 뒤,
saveOutline으로 구간 계획을 저장해.
그다음 바로 1부부터 작성해.

각 단원은 0~11번 형식을 유지해.
이미지 삽입 위치, 교재 보강, 출제양식 기반 판단, 수식 검수, 디자인 검수 항목은 saveProjectItems로 따로 저장해.

최종본이 완성되면 exportProjectNoteAsSource를 호출해서 generated_note로 저장해.
반환된 source_id를 알려줘.

---

# 2. 시험범위 판별

기출 문제와 현재 시험범위를 비교해서 현재 시험범위인 문항과 아닌 문항을 구분해줘.

searchSources로 past_exam, exam_trend, generated_note, textbook을 검색하고,
문항별로 현재 시험범위 / 시험범위 외 / 일부 포함 / 확인 필요로 분류해줘.

---

# 3. 같은 시험지 형식 재현

기존 시험지의 형식과 출제 스타일을 분석해서 현재 시험범위에 맞는 새 시험지를 만들어줘.

문항 수, 난이도 분포, 보기 방식, 계산량, 서술 요구 방식을 기존 시험지와 비슷하게 유지하되,
시험범위 밖 개념은 제외하거나 범위 내 개념으로 변형해줘.
완성 후 SolvePad 문제팩 JSON으로 만들고 saveProblemPack Action을 호출해줘.

---

# 4. 출제 스타일 분석 기반 신규 문제셋

저장된 generated_note, 기출 문제, 시험 경향, 교재를 바탕으로 새로운 예상 문제셋을 만들어줘.

기본 개념 확인, 대표 유형, 조건 변형, 계산 강화, 함정/오개념 점검, 고난도 종합 문제를 섞어줘.
각 문제는 SolvePad JSON 스키마를 지키고, solution에 concepts, actualSolution, cautions, tips를 모두 포함해줘.
완성 후 saveProblemPack Action으로 저장하고 import_url을 알려줘.

---

# 5. CASIO 계산기 프로그램 설계 시작

저장된 generated_note와 textbook을 바탕으로 CASIO fx-9860GIII/fx-9750GIII에서 실행할 계산기 PRGM을 설계해줘.

먼저 searchSources로 generated_note와 textbook 관련 chunk를 검색해.
계산 절차, 입력값, 출력값, 사용 공식, 예외 조건을 정리해.
ANALYZE → PLAN → BLUEPRINT 순서로 진행하고,
validateCalculatorBlueprint로 검증한 뒤 generateCalculatorProgram으로 TXT 파일을 생성해.


---

# 6. 단원 자동 매핑 시작

저장된 교재, 강의자료, 전사본, 시험 문제, generated_note/external_note를 단원별로 매핑해줘.

searchSources로 textbook, lecture_slides, transcript, past_exam, generated_note, external_note를 검색하고,
같은 개념/공식/문제 유형/강의 흐름이 겹치는 항목을 단원 단위로 묶어줘.

각 단원마다 다음을 연결해:
- 교재 페이지
- 강의자료 페이지/슬라이드
- 전사본 시간대 또는 chunk
- 기출 문제 번호
- 정리본 단원
- 시험 관련도
- confidence
- CHECK SOURCE 항목

완성 후 saveUnitMap Action으로 저장하고 unit_map_id를 알려줘.

---

# 7. 외부 정리본 업로드 텍스트 저장

내가 붙여넣는 외부 정리본을 external_note로 저장해줘.

saveExternalNote 또는 saveTextSource Action을 사용하고,
title은 내가 지정한 제목을 사용해.
저장 후 source_id를 알려줘.


---

# 8. 미매핑 자료 모두 매핑

업로드한 자료들 중 매핑 안 된 자료들 모두 매핑 진행해.

listUnmappedSources를 호출해서 미매핑 자료를 확인하고,
searchSources로 관련 chunk를 검색한 뒤,
교재/강의자료/전사본/시험지/정리본을 단원 기준으로 연결해.
완성 후 saveUnitMap으로 저장하고 unit_map_id를 알려줘.

---

# 9. 선택지 보여주기

저장된 자료와 매핑 정보를 읽어서 작업 선택지를 보여줘.

getWorkflowOptions를 호출한 뒤,
과목명 또는 자료 묶음, 선택 가능한 단원, 작업 모드를 정리해서 보여줘.
단원은 여러 개 선택 가능하게 해.
모드는 정리본 / 계산기 / 문제지 / 전사본 보정 / 단원 매핑 중에서 선택하게 해.

---

# 10. 정리본 직전 버전 수정

내가 말한 수정사항을 직전 정리본 버전에 반영해줘.

getLatestNoteVersion으로 직전 버전을 불러오고,
수정사항만 반영해서 새 버전을 만든 뒤,
saveNoteVersion으로 저장해.
기존 버전은 덮어쓰지 마.

---

# 11. 전사본 용어 보정

전사본 내용을 교재와 강의자료 용어 기준으로 보정해줘.

searchSources로 transcript, lecture_slides, textbook을 찾아보고,
잘못 들린 용어, 오탈자, 표기 흔들림을 수정해.
의미는 바꾸지 말고, 불확실한 부분은 CHECK TERM 또는 [확인 필요]로 남겨.
terminology_map과 change_log를 포함해 saveTranscriptRevision으로 저장해.


---

# 12. 강의자료 기준 단원 매핑

강의자료 기준으로 단원 번호를 먼저 매기고, 교재/전사본/시험지/정리본을 그 단원에 확장 매핑해줘.

먼저 lecture_slides를 searchSources로 검색해서 단원 번호, 단원명, slide/page range를 잡아.
그다음 textbook, transcript/corrected_transcript, past_exam, exam_trend, generated_note, external_note를 검색해서 해당 강의자료 단원에 붙여.
강의자료 기준 단원에 연결할 근거가 부족한 자료는 unmapped 또는 extensionOnly로 분리해.
완성 후 saveUnitMap으로 저장해.


---

# 13. 간단 명령 예시

아래 입력은 모두 자동 실행해.

1
→ 미매핑 자료 모두 매핑

2
→ 저장된 자료와 매핑 정보 기준으로 선택지 보여주기

정리 전체
→ 전체 단원 정리본 생성, 버전 저장

정리 1-3
→ 강의자료 기준 1~3단원 정리본 생성, 버전 저장

문제 2
→ 강의자료 기준 2단원 SolvePad 문제팩 생성

계산기 3,4
→ 강의자료 기준 3,4단원 CASIO PRGM 생성

전사본 보정
→ 전사본을 강의자료/교재 용어 기준으로 보정

수정
→ 직전 정리본 버전 조회 후 사용자의 수정사항 반영
