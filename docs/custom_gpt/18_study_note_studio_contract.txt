# Study Note Studio 계약

목적: 저장된 GPT 생성 정리본, 외부 정리본, 시험 직전 정리, 계산기 사용법 문서를 사이트에서 열어 공부하고 직접 수정한다.

## 핵심 UI 원칙
- 사용자는 Markdown 코드를 직접 고치는 방식이 아니라, 실제 출력물 형태의 문서를 보면서 수정한다.
- 본문은 contenteditable 기반의 문서형 편집 영역이다.
- 제목, 문단, 목록, 표, 이미지, 형광펜은 화면에서 실제 형태로 보인다.
- Markdown 원문은 고급 수정용으로만 `원문 보기` 버튼에서 선택적으로 제공한다.
- 이미지는 본문에 실제 이미지 카드로 표시하고, 긴 data URL이나 이미지 주소를 기본 화면에 노출하지 않는다.
- 이미지 카드는 삭제 버튼을 제공한다.

## 지원 기능
- 저장된 정리본 목록 조회
- 실제 출력물 형태에서 키보드로 텍스트 직접 수정
- 텍스트 드래그 선택 후 형광펜 표시
- 이미지 파일 삽입 및 삭제
- 표 렌더링 및 표 형태 유지
- 저장: updateStudyNote 호출
- 새 문서 저장: saveStudyNote 호출
- Markdown 다운로드
- Word 다운로드
- PDF 저장: 브라우저 인쇄/저장 사용

## Markdown 생성 규칙
- 제목은 #, ##, ###를 우선 사용한다.
- 표는 표준 Markdown 표 형식으로 작성한다. 예:

| 항목 | A | B |
|---|---|---|
| 원리 | 내용 | 내용 |

- 표 앞뒤에는 빈 줄을 둔다.
- 비어 있는 항목은 `없음`이라고 쓴다.
- `M0`, `N/A`, `TBD` 같은 임시 토큰을 쓰지 않는다.

## 수식 규칙
- 내부 저장은 Markdown/LaTeX 원문을 유지한다.
- 화면에서는 MathJax로 수식을 렌더링한다.
- 수식 노드는 기본적으로 편집 중 깨지지 않도록 보호한다.
- 수식 자체를 고쳐야 할 때는 `원문 보기`에서 `$...$`, `$$...$$`, `\(...\)`, `\[...\]`를 수정한다.
- 분수는 `\frac{}{}`를 사용한다.
- 행렬은 `bmatrix`, `pmatrix`, `vmatrix`를 사용한다.
- 수식은 이미지로 변환하지 않는다.

## Word/PDF
- Word 다운로드는 `/study/notes/{source_id}/download.docx`를 사용한다.
- PDF는 `/study/notes/{source_id}/print`에서 브라우저 인쇄 → PDF 저장을 사용한다.

## GPT 사용 규칙
- 정리본 생성 결과는 Markdown 원문으로 저장하되, 사용자는 Study Note Studio에서 실제 문서 형태로 편집한다고 안내한다.
- 수정 요청 시 기존 source_id가 있으면 updateStudyNote로 교체한다.


[Study Note 렌더링 안정성]
- Study Note에서 일반 설명이나 흐름을 코드블록으로 만들면 문서 가독성이 떨어진다.
- 정리본 생성 시 검은 코드블록처럼 보일 수 있는 fenced code block을 일반 설명용으로 사용하지 않는다.
- 이미지 삽입 후보 목록을 본문에 길게 넣지 않는다. 이미지가 필요한 경우 본문 마지막 검수 단계에서 적절한 위치에 실제 이미지 또는 짧은 이미지 카드만 삽입한다.
