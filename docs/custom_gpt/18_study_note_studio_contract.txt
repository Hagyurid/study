# Study Note Studio 계약 v20

목적:
저장된 GPT 생성 정리본, 외부 정리본, 시험 직전 정리, 계산기 사용법 문서를 사이트에서 열어 공부하고 직접 수정한다.

사이트:
- `/static/study/index.html`

핵심 UI 원칙:
- 사용자는 Markdown 코드를 직접 고치는 방식이 아니라, 실제 출력물 형태의 문서를 보면서 수정한다.
- 본문은 contenteditable 기반의 워드형 편집 영역이다.
- 제목, 문단, 목록, 이미지, 형광펜은 화면에서 실제 형태로 보인다.
- Markdown 원문은 고급 수정용으로만 `원문 보기` 버튼에서 선택적으로 제공한다.
- 이미지는 본문에 실제 이미지 카드로 표시하고, 긴 data URL이나 이미지 주소를 기본 화면에 노출하지 않는다.

지원 기능:
- 저장된 정리본 목록 조회
- 실제 출력물 형태에서 키보드로 텍스트 직접 수정
- 텍스트 드래그 선택 후 형광펜 표시
- 이미지 파일 삽입 및 화면 내 이미지 카드 표시
- 저장: updateStudyNote 호출
- 새 문서 저장: saveStudyNote 호출
- Markdown 다운로드
- Word 다운로드
- PDF 저장: 브라우저 인쇄/저장 사용

수식 규칙:
- 내부 저장은 Markdown/LaTeX 원문을 유지한다.
- 화면에서는 MathJax로 수식을 렌더링한다.
- 수식 노드는 기본적으로 편집 중 깨지지 않도록 보호한다.
- 수식 자체를 고쳐야 할 때는 `원문 보기`에서 `$...$`, `$$...$$`, `\(...\)`, `\[...\]`를 수정한다.
- 분수는 `\frac{}{}`
- 행렬은 `bmatrix`, `pmatrix`, `vmatrix`
- 수식은 이미지로 변환하지 않는다.

Word/PDF:
- Word 다운로드는 `/study/notes/{source_id}/download.docx`를 사용한다.
- 한글 제목 파일명도 안전하게 처리해야 한다.
- PDF는 `/study/notes/{source_id}/print`에서 브라우저 인쇄 → PDF 저장을 사용한다.

GPT 사용 규칙:
- 정리본 생성 결과는 Markdown 원문으로 저장하되, 사용자는 Study Note Studio에서 실제 문서 형태로 편집한다고 안내한다.
- 수정 요청 시 기존 source_id가 있으면 updateStudyNote로 교체한다.
- 사용자가 “사이트에서 공부”, “정리본 열기”, “Study Note”라고 하면 Study Note Studio 링크를 안내한다.


## v21 WYSIWYG 우선 규칙

Study Note Studio는 Markdown 원문 편집기가 아니라 실제 출력물 형태를 보면서 수정하는 문서형 편집기다.

- 일반 텍스트는 본문에서 직접 클릭해 수정한다.
- 텍스트를 드래그해서 형광펜을 적용한다.
- 이미지는 화면에 실제 이미지 카드로 표시하고 긴 data URL은 사용자에게 노출하지 않는다.
- Markdown 원문은 수식이나 내부 구조를 복구할 때만 고급 기능으로 연다.
- Word 다운로드 오류가 있으면 배포 버전과 `/study/notes/{source_id}/download.docx`를 확인한다.
