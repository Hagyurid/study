정리본 작성 규칙 v23.7 NO_GRAPH_IMAGE

목적
- 단원별 Study Note 정리본을 텍스트, LaTeX 수식, Markdown 표 중심으로 작성한다.
- 그래프·그림·이미지 자동 생성 또는 첨부는 하지 않는다.
- 그래프가 필요한 경우 실제 이미지 대신 개형, 축, 기울기, 임계점, 의미를 텍스트로 설명한다.

# 기본 규칙
- 단원 번호와 순서는 `lecture_slides` 기준.
- 전사본은 교수 설명 흐름과 강조점 복원에 사용.
- 교재는 개념·용어·공식·예시 보강에 사용.
- 시험정보/기출은 시험 포인트 반영에 사용.
- 자료에 없는 내용은 임의 보강 금지.
- 불확실하면 `[확인 필요]`, `CHECK SOURCE`, `CHECK FORMULA`, `CHECK CONDITION`.

# 작성 규칙
- 개념은 정의 → 필요성 → 식/조건 → 적용 → 시험 포인트 순서.
- 식은 변수 뜻, 단위, 조건, 예외, 부호·첨자 주의 포함.
- 계산 예시는 식 → 대입값 → 계산 결과 → 해석 순서.
- 그래프는 이미지로 넣지 말고 텍스트로 설명한다.
- 그래프 설명에는 x축, y축, 증가/감소, 기울기, 임계점, 의미를 포함한다.
- 수치값, 변수, 비교, 조건은 일반 Markdown 표로 정리한다.
- `GRAPH_IMAGE`, `VALUE_TABLE`, `SLIDE_IMAGE` 마커를 자동 작성하지 않는다.
- 그림이 꼭 필요하면 `[이미지 확인 필요: 설명]`으로 남긴다.
- 수식 줄바꿈에 `<br>`, `< br >`, `&lt;br&gt;`를 쓰지 않는다.
- 저장 전 누락된 단원, 식, 단위, 확인 필요 표시를 검수한다.
