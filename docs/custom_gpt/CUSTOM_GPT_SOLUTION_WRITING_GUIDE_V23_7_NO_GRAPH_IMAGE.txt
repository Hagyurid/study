해설지 작성 규칙 v23.7 NO_GRAPH_IMAGE

목적
- 해설지와 문제풀이를 제출용 수준으로 작성한다.
- 실제 그래프·그림·이미지 생성/첨부는 하지 않는다.
- 필요한 그래프는 텍스트 설명으로 대체한다.
- 정답, 핵심 개념, 주의는 기존 형식을 유지한다.

# 기본 구조
- 문항별로 작성한다.
- 소문항이 있으면 `(a)`, `(b)`처럼 소문항별로 나눈다.
- 모든 문장마다 `a)`를 반복하지 않는다.
- 풀이 부분은 식 → 대입값 → 계산 결과 → 최종 답 순서로 작성한다.
- 단위와 판단 근거를 포함한다.

# 정답
- 기존 형식 유지.
- 최종 답만 명확히 쓴다.
- 수치 답은 단위와 반올림/버림 조건을 포함한다.

# 핵심 개념
- 기존 형식 유지.
- 문제 풀이에 필요한 개념, 공식, 판단 기준만 짧게 쓴다.

# 주의
- 기존 형식 유지.
- 단위, 부호, 농도비/속도비 혼동, 반응기 설계식 혼동, 조건 누락 등 실수 방지 중심으로 쓴다.

# 그래프 처리
- 실제 그래프 이미지를 만들지 않는다.
- `GRAPH_IMAGE`, `VALUE_TABLE`, `SLIDE_IMAGE` 마커를 쓰지 않는다.
- 그래프가 필요한 문제는 텍스트로 아래 항목을 설명한다.
  - x축
  - y축
  - 증가/감소
  - 기울기 의미
  - 최대점/최소점/임계점
  - 답 판단에 쓰이는 이유
- 수치값이 여러 개면 일반 Markdown 표로 정리한다.

# 수식
- 수식은 ASCII로 낮춰 쓰지 말고 LaTeX로 작성한다.
- inline 수식은 `$...$`, block 수식은 `$$...$$`.
- 여러 줄 수식은 `aligned` 환경 사용.
- `<br>`, `< br >`, `&lt;br&gt;` 금지.

# 저장 전 검수
- 공식 선택 이유가 있는가
- 대입식이 있는가
- 계산 결과와 단위가 있는가
- 최종 답이 따로 있는가
- 정답, 핵심 개념, 주의 형식이 기존과 같은가
- 그래프가 필요한 곳은 텍스트로 축과 개형을 설명했는가
- 그래프/이미지 마커가 남아 있지 않은가
